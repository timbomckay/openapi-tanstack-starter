## Common Patterns

### Adding a new route

Create a file in `src/routes/`. The file name becomes the URL path.

| File                             | URL                |
| -------------------------------- | ------------------ |
| `src/routes/orders/index.tsx`    | `/orders`          |
| `src/routes/orders/$orderId.tsx` | `/orders/:orderId` |
| `src/routes/orders/new.tsx`      | `/orders/new`      |

Boilerplate:

```tsx
import { createFileRoute } from '@tanstack/react-router';

export const Route = createFileRoute('/orders/')({
  component: OrdersPage,
});

function OrdersPage() {
  return <div>Orders</div>;
}
```

TanStack Router Vite plugin regenerates `routeTree.gen.ts` on save automatically.

### Using a Hey API query

Each API has its own generated client. **Always pass `client` explicitly** so the right API is called:

```tsx
import { useQuery } from '@tanstack/react-query';
import { getPetByIdOptions } from '@/api/petstore/generated/@tanstack/react-query.gen';
import { petstoreClient } from '@/api/petstore/client';

// Inside a component:
const { data, isLoading, isError } = useQuery(
  getPetByIdOptions({
    client: petstoreClient, // ← always required with multiple APIs
    path: { petId: 42 }, // path params
    query: { status: 'available' }, // query params
  }),
);
```

### Using a Hey API mutation

```tsx
import { useMutation, useQueryClient } from '@tanstack/react-query';
import { addPetMutation } from '@/api/petstore/generated/@tanstack/react-query.gen';
import { petstoreClient } from '@/api/petstore/client';

const queryClient = useQueryClient();
const mutation = useMutation({
  ...addPetMutation({ client: petstoreClient }),
  onSuccess: () => {
    void queryClient.invalidateQueries({
      queryKey: [{ _id: 'findPetsByStatus' }],
    });
  },
});

// Call:
mutation.mutate({
  body: { name: 'Buddy', photoUrls: [], status: 'available' },
});
```

### Building a form (TanStack Form + Zod)

We do NOT use `react-hook-form`. Forms use **TanStack Form** with Zod for validation,
following the [form composition pattern](https://tanstack.com/form/latest/docs/framework/react/guides/form-composition).

#### Form composition infrastructure

Hook/context plumbing lives in `src/hooks/`, UI field components in `src/components/form/fields/`:

| File                                          | Purpose                                                               |
| --------------------------------------------- | --------------------------------------------------------------------- |
| `src/hooks/form-context.ts`                   | Creates `fieldContext` / `formContext` via `createFormHookContexts()` |
| `src/hooks/use-app-form.ts`                   | Exports `useAppForm` and `withForm` via `createFormHook()`            |
| `src/components/form/fields/text-field.tsx`   | Reusable text input — uses `useFieldContext<string>()` internally     |
| `src/components/form/fields/select-field.tsx` | Reusable select — uses `useFieldContext<string>()` internally         |

To add a new reusable field type, create it in `src/components/form/fields/`, call
`useFieldContext<T>()` for field state, then register it in `src/hooks/use-app-form.ts`.

#### Using forms

```tsx
import { useAppForm } from '@/components/form/use-app-form';
import { z } from 'zod';
import { Button } from '@/components/ui/button';

const schema = z.object({
  name: z.string().min(1, 'Required'),
  status: z.enum(['active', 'inactive']),
});

// ⚠ TanStack Form v1: do NOT pass type param — types are inferred from defaultValues
function MyForm() {
  const form = useAppForm({
    defaultValues: { name: '', status: 'active' as const },
    onSubmit: async ({ value }) => {
      const result = schema.safeParse(value);
      if (!result.success) return;
      // submit result.data
    },
  });

  return (
    <form
      onSubmit={(e) => {
        e.preventDefault();
        void form.handleSubmit();
      }}
    >
      {/* AppField wraps form.Field and passes registered field components to the render prop */}
      <form.AppField
        name="name"
        validators={{
          onChange: ({ value }) => {
            const r = schema.shape.name.safeParse(value);
            return r.success ? undefined : r.error.issues[0]?.message;
          },
        }}
      >
        {(field) => <field.TextField label="Name" required />}
      </form.AppField>

      <form.AppField name="status">
        {(field) => (
          <field.SelectField
            label="Status"
            items={[
              { value: 'active', label: 'Active' },
              { value: 'inactive', label: 'Inactive' },
            ]}
          />
        )}
      </form.AppField>

      <Button type="submit">Submit</Button>
    </form>
  );
}
```

#### How it works

- `useAppForm` is identical to `useForm` but returns a `form` augmented with `AppField` and `AppForm`.
- `form.AppField` behaves like `form.Field` but its render-prop child receives the field API **extended with all registered field components** (`field.TextField`, `field.SelectField`, …).
- Each field component calls `useFieldContext()` internally to read field state — no prop-drilling.
- `withForm` is available for splitting large forms into sub-components while keeping full type safety.

### Navigation buttons (Link styled as Button)

The `base-nova` shadcn style uses **Base UI** primitives which do NOT support `asChild`. Use `buttonVariants` + TanStack Router `Link` instead:

```tsx
import { Link } from '@tanstack/react-router';
import { buttonVariants } from '@/components/ui/button';

// ✅ Correct — Link with button styling
<Link to="/pets" className={buttonVariants({ variant: 'ghost', size: 'sm' })}>
  Back
</Link>;

// ❌ Wrong — asChild does not exist on the Base UI Button
// <Button asChild><Link to="/pets">Back</Link></Button>
```

## Conventions

| Topic                   | Convention                                                                                            |
| ----------------------- | ----------------------------------------------------------------------------------------------------- |
| File names              | `kebab-case.tsx`                                                                                      |
| Component names         | `PascalCase`                                                                                          |
| Exported route constant | Always named `Route`                                                                                  |
| Path alias              | `@/` maps to `src/`                                                                                   |
| API imports             | Import query options from `@/api/<name>/generated/@tanstack/react-query.gen`                          |
| API client              | Each API has `src/api/<name>/client.ts`. Always pass `client: <name>Client` in query/mutation options |
| Zod schemas             | Define at module level, derive types with `z.infer<typeof schema>`                                    |
| shadcn components       | Never hand-edit `src/components/ui/`                                                                  |
| Generated files         | Never hand-edit files ending in `.gen.ts`                                                             |
| No `any`                | TypeScript strict mode; avoid `any`, use `unknown` when needed                                        |
| Toast notifications     | Use `toast()` from `sonner` (`import { toast } from 'sonner'`)                                        |

## Changing an existing API's spec

1. Update `input` for the relevant entry in `openapi-ts.config.ts`
2. Update `baseUrl` in `src/api/<api-name>/client.ts`
3. Run `npm run api:generate` — the `src/api/<api-name>/generated/` directory is fully replaced

## Adding a new API

1. **Config** — uncomment (or add) an entry in `openapi-ts.config.ts`, set `input` and `output.path`
2. **Client** — copy `src/api/nba/client.ts`, uncomment and update `baseUrl`, rename the export
3. **Generate** — run `npm run api:generate`
4. **Wire up** — add `import '@/api/<name>/client'` to `src/main.tsx`
5. **Use in routes** — import from `@/api/<name>/generated/@tanstack/react-query.gen` and pass the client explicitly
